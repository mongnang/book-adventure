## 기준 상태

- 작업 시작 시 작업 트리는 깨끗했습니다.
- 실제 배포는 `.github/workflows/azure-static-web-apps-calm-beach-0aaa31800.yml`에서 정적 앱 `my-webapp`, API `my-webapp/api`를 사용합니다.
- 배포된 `/api/health`에서 Azure OpenAI Responses API v1, `gpt-5-mini`, Cosmos DB, 교사 결과 코드 설정이 활성화된 것을 확인했습니다.
- 로컬 실행은 `file://`가 아니라 HTTP 서버 또는 `swa start my-webapp --api-location my-webapp/api`를 사용합니다.

## 1. OpenAI 속도와 안전한 시간 기록

- 일반 독서 답변, 인물 대화, 활동 1 프롬프트의 출력 상한을 800토큰으로 맞췄습니다.
- `reasoning_effort: "minimal"`, `verbosity: "low"` 및 Responses API의 `reasoning.effort`, `text.verbosity`, `max_output_tokens` 형식을 자동 시험했습니다.
- 질문과 선택 인물에 맞춰 단서 후보를 점수화하고 상위 2개만 프롬프트에 넣습니다.
- Azure OpenAI, Cosmos DB, 전체 처리 시간을 나눠 기록하며 앱 요청 ID와 OpenAI 요청 ID를 연결합니다.
- 로그는 단계, API 경로, HTTP 상태, 오류 코드, 안전하게 줄인 메시지와 시간만 기록합니다. 프롬프트 원문, 학생 정보, 키와 토큰은 기록하지 않습니다.
- 출력이 `incomplete`, `length`, `max_output_tokens`로 끝나면 성공으로 오인하지 않고 잘린 응답 오류로 처리합니다.

기준 배포(이번 미커밋 변경 전)에서 합성 학생으로 직접 측정한 API 시간:

| 요청 | 전체 관측 시간 | 결과 |
| --- | ---: | --- |
| 일반 독서 답변 | 5,949ms | 완결된 한국어 답변 |
| 인물 대화 | 2,036ms | 응답은 완결됐지만 선택 인물과 다른 인물로 답하는 회귀 발견 |
| 활동 1 이미지 프롬프트 | 10,837ms | API 및 저장 성공. 시험 도구의 문자 인코딩 영향으로 품질 판정 보류 |

인물 혼동은 선택 인물 이름과 관점을 전 구간 유지하도록 프롬프트를 보강했습니다. 현재 변경의 배포 후 재측정은 브랜치 커밋·푸시가 막혀 아직 수행하지 못했습니다.

## 2. 활동 1 API 오류와 제출 저장

- 브라우저 요청에 45초 제한과 요청 ID를 추가했습니다.
- 네트워크, HTTP, 시간 초과, JSON, 데이터, 서버 내부, Azure OpenAI 오류를 서로 다른 코드와 단계로 분류합니다.
- 학생 화면은 오류 종류에 맞는 짧은 안내와 같은 버튼으로 다시 시도하는 방법을 표시합니다.
- 프롬프트 생성 실패 후 결과 화면으로 잘못 넘어가거나 빈 프롬프트를 읽는 오류를 막았습니다.
- 제출 API는 Cosmos 저장 시간, 전체 시간, 저장 성공 여부를 반환합니다. 서버 저장 실패 시 기존 브라우저 임시 저장 동작은 유지합니다.

## 3. 교사 이미지 생성 프롬프트 탭

- `titleScenarioSubmission` 기록만 모으는 별도 탭을 추가했습니다.
- 날짜별 그룹과 날짜 필터, 학생·책·제출 시각·전체 프롬프트 표시를 구현했습니다.
- 개별/다중/현재 목록 전체 선택, 선택 해제, 클립보드 복사와 UTF-8 BOM TXT 내보내기를 구현했습니다.
- 복사/TXT는 `제출 일시`, `학생`, `책`, `이미지 생성 프롬프트` 순서로 항목을 구분합니다.
- 데이터는 교사 API가 Cosmos DB에서 다시 읽으므로 서버 저장에 성공한 제출은 새로고침 뒤 유지됩니다.

## 4. 레이아웃과 스크롤

- 첫 제목을 두 개의 줄바꿈 단위로 묶어 `모험`이 글자 중간에서 갈라지지 않게 했습니다.
- 활동 2 대화창의 오른쪽을 정보 패널 앞 16px까지 넓혔고 좁은 화면에서는 전체 폭으로 전환합니다.
- 활동 3 인물 목록에 화면 높이 기반 영역, 내부 세로 스크롤, 마지막 카드 여백, 터치 스크롤과 키보드 포커스를 적용했습니다.
- 활동 3 메시지 영역을 독립 스크롤로 만들고 입력 영역은 고정했습니다.
- 사용자가 바닥에서 64px 이내일 때만 새 메시지로 이동하고, 위 대화를 읽는 중이면 기존 위치를 유지합니다.
- 스크롤바는 짙은 갈색 트랙과 금빛 손잡이, 12px 폭과 둥근 모서리를 사용합니다.

## 5. 책 데이터와 파일 정리

- 전체 책 배열은 `books.json`에만 남기고 JavaScript에는 공통 표시 기본값만 유지했습니다.
- JSON을 읽지 못하면 빈 책장과 오류 안내, 다시 시도 버튼을 표시하며 오래된 복사본을 조용히 표시하지 않습니다.
- 향후 추가할 표지·인물 이미지 경로와 모든 이미지 파일은 사용자 요청에 따라 그대로 보존했습니다.
- 삭제/사용/보류 분류와 근거는 `FILE_CLEANUP_AUDIT.md`에 기록했습니다.
