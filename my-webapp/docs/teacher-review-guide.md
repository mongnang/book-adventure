# 교사용 학생별 결과판 사용법

작성일: 2026-07-10

교사용 결과판은 별도 페이지가 아니라 학생 앱 안에 숨겨진 결과실로 들어갑니다.
학생 정보 입력 화면 아래쪽의 작은 `교사용 결과 확인` 버튼을 누른 뒤 교사용 코드를 입력하면 열립니다.

## 핵심 원칙

- 학생 구분 기준은 `반 + 번호`입니다.
- 학생 결과는 반과 번호를 기준으로 묶습니다.
- 닉네임은 학생 식별값이 아니라 해당 참여 회차의 활동용 이름으로 저장됩니다.
- 같은 학생이 여러 번 활동해도 기존 기록을 덮어쓰지 않고 시간순으로 누적합니다.
- 저장되는 각 활동은 별도 기록입니다.

## 화면 열기

배포 뒤에는 학생 앱의 일반 주소로 들어갑니다.

```text
https://정적앱주소/
```

로컬에서 확인할 때도 기존 앱 파일을 엽니다.

```text
C:\Users\Lee\Desktop\my-webapp\index.html
```

## 교사용 코드 설정

교사용 결과판이 실제 Cosmos DB 기록을 읽으려면 Azure Static Web Apps 환경 변수에 아래 값을 추가해야 합니다.

```text
TEACHER_ACCESS_CODE
```

예시는 다음과 같습니다.

```text
TEACHER_ACCESS_CODE = 선생님만-아는-긴-코드
```

이 값은 비밀값입니다. 코드 파일이나 문서에 실제 값을 적지 않습니다.

## 화면 사용 순서

1. 앱의 `학생 정보 입력` 화면으로 갑니다.
2. 입력 폼 아래쪽의 작은 `교사용 결과 확인` 버튼을 누릅니다.
3. 비밀 문 화면에서 `교사용 코드`를 입력하고 `문 열기`를 누릅니다.
4. 코드가 맞으면 다이얼이 열리는 연출 뒤 `학생별 결과판`으로 이동합니다.
5. 왼쪽 학생 목록에서 학생을 선택합니다.
6. 오른쪽에서 `전체`, `활동 1`, `활동 2`, `활동 3`, `입장 기록` 탭 중 하나를 선택합니다.
7. 선택한 탭의 활동 기록을 시간순으로 확인합니다.
8. 활동 탭 아래의 `참여별 점수 변화` 그래프에서 회차별 평가 총점을 확인합니다.
9. 필요하면 `현재 탭 CSV`를 눌러 지금 보고 있는 탭의 기록만 내려받습니다.
10. 학생의 전체 결과를 공유하거나 인쇄하려면 `학생 HTML 보고서`를 내려받습니다.

## Cosmos DB에서 복사해서 붙여넣는 절차는 이제 하지 않습니다

이전에는 Azure Portal의 Cosmos DB Data Explorer에서 SQL을 실행한 뒤 결과 JSON을 복사해서 확인하는 방식이 필요했습니다.
이제는 앱 안의 교사용 결과실이 같은 데이터를 API로 직접 읽어 오므로, 선생님이 Cosmos DB 결과를 복사해서 붙여넣지 않아도 됩니다.

평상시 절차는 아래처럼 바뀝니다.

1. 학생들이 앱에서 활동을 진행합니다.
2. 활동 기록은 서버 API를 통해 Cosmos DB에 저장됩니다.
3. 선생님은 앱의 `학생 정보 입력` 화면에서 `교사용 결과 확인`을 누릅니다.
4. 비밀 문 화면에서 교사용 코드를 입력합니다.
5. 코드가 맞으면 앱이 `/api/adventure/teacher-results`를 호출합니다.
6. 서버가 Cosmos DB 기록을 읽고, 학생을 `반 + 번호` 기준으로 묶어 돌려줍니다.
7. 선생님은 결과실에서 학생별 기록과 활동별 탭을 확인합니다.
8. 필요한 경우 `현재 탭 CSV` 또는 `학생 HTML 보고서`로 내려받습니다.

Cosmos DB Data Explorer는 이제 문제 해결용으로만 사용합니다.
예를 들어 기록이 저장되지 않는 것처럼 보일 때 아래 쿼리로 원본 저장 여부를 확인할 수 있습니다.

```sql
SELECT * FROM c WHERE c.type IN ("titleScenarioSubmission", "questionTurn", "conversationAssessment", "characterChatTurn")
```

즉, 수업 운영 흐름은 `Cosmos DB에서 복사 → 붙여넣기`가 아니라 `교사용 코드 입력 → 결과실 자동 조회`입니다.

## 학생별 세부 탭

학생 상세 화면은 활동 기록을 한 줄에 모두 섞어 보여주지 않습니다.

- `전체`: 해당 학생의 모든 기록을 시간순으로 봅니다.
- `활동 1`: 책 제목 상상하기 관련 대화와 제출만 봅니다.
- `활동 2`: 질문, 정답 확인, 평가 기록만 봅니다.
- `활동 3`: 학생이 선택한 인물, 학생의 말, 인물의 답변만 봅니다.
- `입장 기록`: 학생 정보 입력 및 세션 시작 기록만 봅니다.

학생을 처음 선택하면 활동 1 기록이 있는 경우 `활동 1` 탭을 먼저 보여줍니다.
활동 1 기록이 없고 활동 2 기록만 있으면 `활동 2` 탭을 먼저 보여줍니다.
활동 1과 활동 2 기록이 없고 활동 3 기록만 있으면 `활동 3` 탭을 먼저 보여줍니다.

각 탭의 기록은 `1번째 참여`, `2번째 참여`처럼 참여 회차별로 묶입니다.
회차 제목을 누르면 해당 참여 기록을 열거나 닫을 수 있고, 가장 최근 참여는 처음부터 펼쳐집니다.
각 회차에는 그때 학생이 활동에서 사용한 닉네임이 함께 표시됩니다.

## 참여별 점수 변화 그래프

- 그래프는 활동 탭과 참여 회차 기록 사이에 표시됩니다.
- 평가 총점은 15점 기준으로 맞춰 `1번째`, `2번째` 순서로 이어집니다.
- `전체`와 `활동 2` 탭에서 평가 점수 변화를 확인할 수 있습니다.
- 활동 1, 활동 3, 입장 기록처럼 점수 평가가 없는 탭에서는 안내 문구가 표시됩니다.

## 학생 HTML 보고서

`학생 HTML 보고서`는 선택한 학생의 전체 참여 기록과 점수 변화 그래프를 하나의 HTML 파일로 내려받습니다.
인터넷 연결 없이 열 수 있으며, 보고서 위쪽의 `인쇄 또는 PDF 저장` 버튼으로 종이 출력이나 PDF 저장을 할 수 있습니다.

## 활동 2 평가 항목

활동 2의 독서 모험 결과는 15점 만점이며 다음 세 항목을 각각 5점으로 평가합니다.

- `질문 태도`: 질문을 이어 가며 인물과 장소의 단서를 확인했는가
- `추리 결론`: 최종 결론이 작품의 핵심 관계와 맞는가
- `합리적 추론`: 정답을 맞힐 때 작품의 장면, 인물의 행동, 장소 단서 등 적합한 근거를 함께 제시했는가

## 표시되는 기록

- 입장 기록
- 활동 1 대화
- 활동 1 제출
- 활동 2 질문
- 활동 2 정답 확인
- 활동 2 평가
- 활동 3 인물 대화

## 저장 방식

서버는 활동 기록을 저장할 때마다 새 ID를 만들어 Cosmos DB에 추가합니다.
따라서 같은 학생이 다시 활동해도 이전 제출물이나 평가 기록을 바꾸지 않고 새 기록으로 쌓습니다.

새로 저장되는 기록에는 아래 학생 구분 필드가 함께 들어갑니다.

```text
studentKey = class:{반}|number:{번호}
studentClassName
studentNumber
studentNickname
```

기존에 저장된 기록에 `studentKey`가 없어도 교사용 결과 API가 `student.className`과 `student.number`를 보고 같은 방식으로 묶습니다.

## 확인용 상태 주소

배포 후 아래 주소에서 설정 여부를 확인할 수 있습니다.

```text
https://정적앱주소/api/health
```

정상이라면 응답에 아래처럼 보입니다.

```json
{
  "teacherReview": {
    "configured": true
  }
}
```
